# CodeDB Integration Contracts

Source: PRD section 16.

| Surface | Contract | Forbidden crossing |
|---|---|---|
| Codex | Use CLI/MCP bounded table outputs. Absolute binary paths may be generated by envctl. | No auth/session hacks. No unbounded raw source by default. |
| Nushell | Nu plugin returns structured tables and honors plugin protocol compatibility. | Do not assume host Nu and Yazelix Nu share registry/protocol. |
| Yazelix | Hosts the CLI/plugin as a runtime tool and generated init/extern bridge under state paths. | Do not edit tracked Yazelix `nushell/config/config.nu`. |
| meta | Supplies repo/project selection and graph context. | Do not replace meta or mutate multi-repo state without gate. |
| envctl | Consumes exported rows/checksums/env facts. | Do not read redb internals directly. |
| runner/fxrun | Owns release proof execution and raw logs. | Do not claim release readiness without runner evidence. |
| GitKB | Stores durable decisions and handoffs. | Do not use chat/session memory as release truth. |
| RTK | May summarize but raw logs remain source of truth. | Compression never replaces failure evidence. |
| Kache/wild/Fenix | Toolchain/cache/linker context must be captured as facts. | Do not confuse toolchain optimization with crate truth. |

## Integration acceptance

An integration is valid only when it has: owner, input rows, output rows, validation gate, forbidden actions, and raw-log/evidence path.

## Extracted NotebookLM Source Alignment

The NotebookLM `Agentic Mindmap` artifact
`a033f801-5ddb-4b34-af1d-36d7288fe11c` is an architecture overview, not
implementation proof. The cross-reference corpus is the locally extracted
planning-spine registry and source set under
`../lifeos/planning-spine-v0/generated/notebooklm_source_{registry.source.csv,extracts/}`.
The checked source tree, exact dependency receipts, and executable tests remain
authoritative when an extracted statement differs from implementation.

| Extracted sources | CodeDB integration contract |
|---|---|
| `NBSOURCE-003`, `NBSOURCE-030` | redb and PostgreSQL remain backend-neutral exact-byte CodeDB stores. RuVector embeddings or semantic indexes are derived layers and are not substitutes for source bytes, receipts, or deterministic materialization. No live CodeDB RuVector adapter is claimed until runtime proof exists. |
| `NBSOURCE-029`, `NBSOURCE-030` | The verified native chain is the local `ruvector-core` Rust crate -> `ruvector-node` `cdylib` -> napi-rs -> platform `.node` npm package -> `@ruvector/core` wrapper -> exact Bun install -> CodeDB capture. The extracted "C++ extension" wording is superseded by the checked Rust `cdylib` and napi-rs sources. Capture preserves the complete locked online Node dependency set, package aliases, and observed wrapper/native versions rather than collapsing them to one package name. |
| `NBSOURCE-031` | envctl consumes exported rows, checksums, and materialized paths. It must not read redb/PostgreSQL internals or silently become CodeDB's storage owner. |
| `NBSOURCE-032` | The Nushell integration remains a native Rust Nu plugin using the plugin protocol's MessagePack serializer. It does not become a shell-script parser or inherit an unverified host registry. |

Whole-tree capture includes regular files and symlink metadata, including safe
Bun `.bin` links. Replay must verify digests, reject absolute or escaping link
targets, and preserve exact-byte capture separately from any lossy AST,
embedding, or graph projection. A non-UTF-8 link target fails closed as an
explicit capture gap instead of being rewritten through a lossy string.

## GitKB Boundary

GitKB stores durable explanations, decisions, and handoffs. It is not the source
of Rust/crate truth, not a raw-source store, and not sufficient release proof by
itself.

CodeDB should publish or link these GitKB-facing facts:

| Row/export | Purpose |
|---|---|
| `codedb_doctrine_summary` | Stable summary of CodeDB authority boundaries and unsupported claims |
| `capture_gaps` | Known incomplete compiler-observable facts |
| `validation_errors` | Malformed, unreadable, ambiguous, or policy-invalid facts |
| `meta_repo_selection` | Repo/project label and explicit scan root |
| `runner_proof_manifest` | Runner-owned release proof rows and raw log paths |
| command-ledger references | Raw command/log evidence without copying raw source blobs |

GitKB may reference raw logs and proof artifacts. It must not embed raw source
blobs, replace runner proof, or turn chat/session memory into release truth.

## RTK Boundary

RTK may summarize long CodeDB outputs for review and handoff. Raw failure logs
remain the source of truth. RTK compression must preserve links to original log
paths and must not replace root-cause evidence.

CodeDB-facing RTK rows or notes should include:

| Row/export | Purpose |
|---|---|
| `raw_log_paths` | Paths to uncompressed command output and failure evidence |
| `runner_proof_manifest.raw_log_path` | Release-gate log references |
| `capture_gaps` summary | Human-sized overview that still points to full rows |
| `validation_errors` summary | Human-sized overview that still points to full rows |

RTK must not hide a failing command behind a successful summary, drop stderr,
rewrite failure causes, or compress away the raw log needed for reproduction.

## Kache, wild, and Fenix Boundary

Kache, wild, and Fenix are environment/toolchain facts for CodeDB context. They
are not CodeDB-owned installs, not substitutes for Cargo/rustc evidence, and not
proof that crate facts are complete.

CodeDB should capture these as facts when relevant:

| Tooling surface | Fact rows |
|---|---|
| Kache | `kache_status`, wrapper path, cache root, enabled/disabled state, provenance source |
| wild | linker path, opt-in feature state, target triple, link args source |
| Fenix | toolchain channel, component list, target list, rustc/cargo/rustfmt/clippy/rust-analyzer paths and versions |

These facts belong in toolchain/context exports that envctl, meta, runner, and
GitKB can consume. CodeDB must not install, upgrade, or repair these tools as a
side effect of scanning. If a fact cannot be observed safely, CodeDB should emit
a capture gap or validation row instead of guessing.

## Ownership Matrix

| Surface | Owner | CodeDB responsibility | Forbidden crossing |
|---|---|---|---|
| GitKB | GitKB project workflow | Store durable explanations, handoffs, task evidence, and proof links | Store raw source blobs or replace runner proof |
| RTK | RTK summarization/compression workflow | Summarize long outputs while preserving raw log paths | Replace failure evidence or erase stderr/root cause |
| Kache | Host/toolchain owner | Capture cache/wrapper facts | Install or tune cache state |
| wild | Host/toolchain owner | Capture linker facts and opt-in state | Treat linker optimization as crate truth |
| Fenix | Host/toolchain owner | Capture toolchain/channel/component/target facts | Install or mutate toolchains |
